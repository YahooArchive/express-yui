The express-yui exports few middleware that provide some basic features to attach information into the res.locals object that could be used to boot YUI in the client runtime.

var express = require('express'),
    expyui = require('express-yui'),
    app = express();

expyui.extend(app);
app.use(expyui.expose());

debug

(
  • config
)
Function public

Defined in lib/middleware.js:153

Forces a request to use yui in debug mode with combine disabled.

// exposing yui into the client side thru state object
app.use(expyui.expose());
// using yui in debug mode when node runs in debug mode with custom filter
if (app.get('env') === 'development') {
    app.use(expyui.debug({filter: 'raw'}));
}

Note: the expyui.debug() middleware only works if it is called after the regular expyui.expose() middleware was called, otherwise it will not be able to overrule the default yui config for the app.

More details about the yui debug mode settings in the YUI API Docs.

Parameters:

    optional debug settings

    • combine Boolean

      default to false

      default to "debug"

expose

() Function public

Defined in lib/middleware.js:113

Exposes YUI into the client side. This middleware bundles expyui.exposeConfig() and expyui.exposeSeed() middleware.

// using it for a mounted middleware
app.use(expyui.expose());

// or using it as a route middleware
app.get('/foo', expyui.expose());

exposeConfig

() Function protected

Defined in lib/middleware.js:44

Exposes the yui configuration into res.locals.state. It also exposes the express-yui wrapper for YUI on the client so you can access app.yui.* on the client side just like you do on the server side. The client wrapper includes app.yui.ready() and app.yui.use() with the corresponding bootstraping code to inject YUI into the page. This middleware will be invoked by expyui.expose() middleware automatically, which means you do not need to call it directly.

exposeSeed

() Function protected

Defined in lib/middleware.js:81

Expose the seed information into res.locals.state. This seed is an array of urls based on the call to app.yui.seed(), which is going to be used by the client bootstrap code to inject YUI into the page. This middleware will be invoked by expyui.expose() middleware automatically, which means you do not need to call it directly.
